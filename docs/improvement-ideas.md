# study-hub 보강 후보 리스트

수강생이 라이브 강의를 놓쳤을 때 자료만 보고도 잘 따라올 수 있도록, 나중에 손볼 수 있는 보강 후보를 정리한 메모입니다.

- 작성일: 2026-05-18
- 작성 동기: 자료 양이 많아 보일 때 비개발자 수강생이 막히지 않게 하려고
- 우선순위는 마지막 섹션 참고

---

## 1. 스크린샷 보강 후보

### Antigravity 설치 페이지 (`/setup/antigravity/`)

| 위치 | 무슨 스크린샷 | 왜 필요한가 |
|------|--------------|------------|
| STEP 1. 다운로드 | antigravity.google/download 페이지 화면 (x64/ARM64 버튼 위치) | "어떤 버튼 눌러야 하지?" 막힘 방지 |
| STEP 2. 보안 경고 | "Windows에서 PC를 보호했습니다" 팝업 + [추가 정보] → [실행] 위치 | 비개발자가 가장 놀라는 순간. 사진 한 장이면 안심 |
| STEP 3. 온보딩 ③ 권한 모드 | Review-driven 선택 화면 | 버전마다 이름 다를 수 있어서 "내 화면이랑 다른데?" 자주 발생 |
| STEP 3. ⑤ Google 로그인 | 로그인 성공 후 Agent Manager 첫 화면 | "이게 뜨면 성공" 확신용 |
| STEP 4. 화면 둘러보기 | 전체 화면에 5군데 영역 표시한 그림 (번호 매겨서) | 글로 "왼쪽 사이드바, 가운데 에디터" 설명 → 그림 한 장이면 한 번에 이해 |
| STEP 5. 한글화 | Extensions 아이콘 위치 + 검색창에 "Korean" 입력한 화면 | "Extensions 아이콘이 뭐 어떻게 생겼지?" 막힘 방지 |

### Claude Code 설치 페이지 (`/setup/claude-code/`)

| 위치 | 무슨 스크린샷 | 왜 필요한가 |
|------|--------------|------------|
| STEP 1. 터미널 열기 | `Ctrl + ``` 누른 후 하단에 열린 터미널 + `PS C:\` 프롬프트 | "어디가 터미널이지? 어떻게 생겼지?" |
| STEP 2. 설치 명령 입력 후 | 설치 진행 중 출력 화면 (점들 진행되는 모습) + 완료 화면 | "이게 잘 되고 있는 건가?" 불안 해소 |
| STEP 2. `claude --version` 결과 | 버전 번호 출력된 화면 | 성공 확인 |
| STEP 3. 첫 실행 로그인 | `claude` 입력 → 브라우저 자동 열림 화면 + "Welcome to Claude Code" 메시지 | 성공 신호 |
| STEP 4. VS Code 확장 | Extensions 검색창에 "Claude Code" + 제작자 Anthropic 표시 부분 | 다른 확장(가짜) 잘못 깔지 않게 |

### 2장 시작 전 준비 (`/tutorial/02-setup/`)

| 위치 | 무슨 스크린샷 | 왜 필요한가 |
|------|--------------|------------|
| 1단계. GitHub 가입 | github.com 메인의 `Sign up` 버튼 위치 + `Continue with Google` 버튼 | 비개발자는 "버튼 어디 있지?" 자주 막힘 |
| 2단계. 이메일 인증 | GitHub Settings → Access → Emails 사이드바 메뉴 화면 | 메뉴 경로 글만 보면 헷갈림 |
| 4단계. 폴더 열기 | Antigravity의 `File → Open Folder` 메뉴 펼친 화면 | 시작 화면 vs 메뉴 분기 안내 보조 |
| 7단계. Node.js 확인 | 터미널에 `node -v` 입력 후 `v24.x.x` 나온 화면 | 성공 모양 확인 |

## 2. 에러 케이스 보강 후보

### Antigravity 설치 페이지

| 단계 | 보강할 에러 케이스 |
|------|------------------|
| STEP 1. 다운로드 | "내 PC가 x64인지 ARM64인지 모르겠어요" — 시스템 종류 확인 못 한 경우. (이미 hint로 "모르겠으면 x64" 안내는 있지만, 확신 위해 한 줄 더) |
| STEP 2. 설치 | 회사 PC라 관리자 권한이 없는 경우 — 설치 안 되면 어떻게 할지 |
| STEP 3. 온보딩 ⑤ 로그인 | "개인 Gmail이 아니면 로그인이 거부됨" 경우 — 어떤 에러 메시지 뜨는지, 어떻게 해결하는지 |
| STEP 3. 끝 | Agent Manager 화면이 안 뜨고 멈춰있는 경우 — 재시작 안내 |
| STEP 5. 한글화 | `Configure Display Language`가 검색해도 안 나오는 경우 — Command Palette 단축키부터 다시 확인 |

### Claude Code 설치 페이지

| 단계 | 보강할 에러 케이스 |
|------|------------------|
| STEP 2. 설치 | ✅ CMD 에러 (이미 추가됨) |
| STEP 2. 설치 | 회사망/방화벽 때문에 `claude.ai/install.ps1` 다운로드가 차단되는 경우 — WinGet이나 수동 설치로 우회 안내 |
| STEP 2. 확인 | `claude --version` 했는데 "명령을 찾을 수 없습니다" — PATH 갱신 안 된 경우 (Antigravity 재시작) |
| STEP 3. 로그인 | Chrome이 자동으로 안 열림 — 수동으로 URL 복사해서 여는 방법 |
| STEP 3. 로그인 | "Welcome to Claude Code"가 안 뜨고 다른 화면이 나옴 — 무료 계정/지역 제한 가능성 안내 |

### 2장 시작 전 준비

| 단계 | 보강할 에러 케이스 |
|------|------------------|
| 1단계. GitHub 가입 | "User name already taken" — 사용자명 중복 처리 안내 |
| 1단계. 가입 | 이메일이 이미 다른 GitHub 계정에 쓰이고 있는 경우 |
| 5단계. git | "git이 설치돼있다는데 `git --version` 했더니 안 됨" — Git for Windows 재설치, PATH 문제 |
| 6단계. gh | `gh auth login` 중 브라우저가 안 열림 — 토큰 방식 로그인 안내 |
| 7단계. Node.js | `npm` 명령은 되는데 `node`만 안 되는 경우 (드물지만 발생) |

### 부록 I에 추가하면 좋은 일반 케이스

- "화면 글자가 너무 작아서 안 보여요" → 확대 단축키 (`Ctrl + +`)
- "실수로 창을 다 닫아버렸어요" → 어떻게 복구하나
- "AI 답변이 영어로 와요" → "한국어로 답해줘" 프롬프트 한 줄

## 3. 우선순위 (만약 나중에 손본다면)

1. **Antigravity STEP 2 보안 경고 스크린샷** — 가장 많이 놀라는 지점
2. **STEP 4 화면 5군데 영역 그림** — 한 장이 글 1페이지를 대신함
3. **Claude Code STEP 2 PowerShell 터미널 모습** — "PS C:\가 뭐예요?" 자주 나옴
4. **GitHub 가입 화면들 (Sign up, Continue with Google)** — 비개발자 첫 진입점
5. **에러 케이스: 회사망 차단 / PATH 갱신 안 됨** — 발생하면 100% 멈추는 케이스

나머지는 수강생 피드백 받고 나서 결정해도 늦지 않음.

---

## 4. 메모

- 자료 자체의 양은 한 달 분량으로 적정. 줄이지 않는 방향으로 결정.
- 라이브 강의로 진행 + 자료는 보조용. 그래도 놓친 사람 대비 보강은 필요.
- 라이브 들으면서 따라오는 비율 ~95%, 자료만 보면 60-70%, 스크린샷 추가 시 80-85% 예상.
