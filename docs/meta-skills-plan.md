## 1. 배경 — 이 작업의 이유

### 1.1 1차 발견 (이전)
스터디 수강생 피드백:
1. 개념 정리 안 됨
2. VS Code UI/UX 낯섦
3. AI 협업 메타스킬 부재 — 캡처 첨부, 좋은 질문 패턴 모름

→ AI 메타스킬이 진짜 핵심이고, 이 스터디의 본질이라는 결론.

### 1.2 2차 발견 (이번 — 더 큰 그림)
- **기존 튜토리얼 순서가 비효율적**
- 환경 세팅 → GitHub 가입 → CLI 설치 → Fork → Clone → 그제서야 라이프 위키 사용
- 사람들이 **최종 결과를 한참 못 봐서** "내가 뭐 하고 있는 건지" 길 잃음
- 막상 도착해서는 이미 지쳐있음

→ **학습 흐름 자체를 재설계해야 함.**

### 1.3 새 학습 흐름 핵심 아이디어

**"결과를 먼저 보여주고, 필요해진 시점에 도구 설치"**

GitHub·CLI는 push할 때만 필요. **남의 레포 가져와서 로컬에서 보고 수정**하는 단계까진 GitHub 가입 + git만 있으면 됨 (gh CLI 로그인은 push 시점에).

### 1.4 운영 이슈 — 점진적 개편 필수

- 22기 수강생이 이미 사이트를 다시보기로 예습·복습 중
- 갑자기 챕터 번호·URL 바뀌면 혼란
- 따라서 **2단계 점진적 개편**:
  - **1단계 (지금, 22기 중)**: 추가만. 메타스킬 트랙 신설 + 기존 페이지 안 건드림
  - **2단계 (23기 시작 전)**: 본격 재배치 + 챕터 번호·파일명 정리
