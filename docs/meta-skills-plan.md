## 1. 배경 — 이 작업의 이유

스터디 수강생 피드백에서 세 가지 문제가 드러남:

1. **개념 정리 안 됨** — 도구/용어가 어떻게 연결되는지 모름
2. **VS Code UI/UX 낯섦** — 메뉴·패널·단축키 못 찾음
3. **AI 협업 메타스킬 부재** — 캡처 첨부, 좋은 질문 패턴 등 AI를 잘 쓰는 법 자체를 모름

`용어집·툴팁`(이미 추가)이 1번을 부분 해결. 하지만 **3번이 진짜 핵심**이라는 결론.

### 발견한 본질

> 툴은 바뀐다 (Antigravity → VS Code 한 달 만에 갈아엎은 게 증거).
> **AI 메타스킬은 안 바뀐다.** 이게 평생 가는 능력이고, 이 스터디의 차별점.

→ **이 스터디의 정체성을 "메타스킬 중심"으로 재정의.** 도구는 그 위에 얹는 시즌 한정 콘텐츠.

## 2. 사이드바 구조 변경

### Before
```
온보딩
환경 세팅
튜토리얼
수강생 사례
```

### After
```
온보딩
환경 세팅
AI 메타스킬   ← 신규 메인 트랙
튜토리얼
수강생 사례
```

**위치 이유**: 환경 세팅 → 메타스킬 → 튜토리얼 흐름. "도구 깔고 → AI한테 잘 묻는 법 익히고 → 실습".

**필요한 작업**:
- `src/components/Sidebar.astro`에 "AI 메타스킬" 그룹 추가
- `src/config/nav.ts`에 메타스킬 섹션 추가 (`NAV_SECTIONS`, `hasSidebar`, `getSectionKey`)
- 메타스킬 페이지 컨테이너: `.page-body` (기존 환경 세팅 톤 재활용)
- 툴팁 시스템: `applyGlossaryTo('.page-body')` 적용

### 사이드바 안 구조 (메타스킬 그룹)

```
AI 메타스킬
  ├ 들어가며 (이 트랙의 의도)
  ├ 1. 막혔을 때 — 스크린샷·오류로 물어보기
  ├ 2. 나만의 학습메이트 만들기 (CLAUDE.md)
  ├ 3. AI와 함께 설계하기 (인터뷰·보기 제시)
  ├ 4. 계획서 받고 진행하기
  ├ (준비 중) 메타프롬프팅
  ├ (준비 중) AI 답변 검증
  ├ (준비 중) 컨텍스트 관리
  └ ...
```

**"준비 중" 처리**:
- 페이지 파일은 만들되 frontmatter status: 초안 또는 메뉴에서 표시는 하되 회색·잠금 아이콘
- 결정 보류 — 첫 페이지 만들고 톤 잡힌 다음 결정

## 3. MVP 4개 — 즉각 효과 큰 것부터

선정 기준: **즉각 효과 + 매일 빈도 + 한 번 익히면 평생**

| # | 페이지 | 통합/단독 |
|---|---|---|
| **1** | **막혔을 때 — 스크린샷·오류로 물어보기** | "모르는 단어 풀어달라" 패턴(8번)을 안에 짧게 추가 |
| **2** | **나만의 학습메이트 만들기 (CLAUDE.md 활용)** | 단독 |
| **3** | **AI와 함께 설계하기 — 인터뷰 + 보기 제시** | 4번(인터뷰) + 5번(보기 제시) 통합 |
| **4** | **계획서 받고 → 검토 → 진행하기** | 단독 |

### 각 페이지의 핵심 메시지

#### 페이지 1: 막혔을 때 — 스크린샷·오류로 물어보기
- **언제 쓰나**: 코드가 안 돌아갈 때, 오류 메시지 떴을 때, 화면이 이상할 때
- **핵심 패턴**: "왜 안 돼" 대신 → 캡처 + 오류 통째로 + 자연어 설명
- **추가**: 모르는 단어 만났을 때 "초보자한테 설명하듯 풀어줘"

#### 페이지 2: 나만의 학습메이트 만들기 (CLAUDE.md)
- **언제 쓰나**: 같은 말 반복 설명하기 귀찮을 때, 내 수준에 맞는 답 받고 싶을 때
- **핵심 패턴**: CLAUDE.md에 "나는 누구인지·어떻게 일하는지" 적어두기
- **참고**: 부록 C `CLAUDE.md 잘 쓰는 법`이 이미 있음 → 거기랑 중복 방지 / 메타스킬 관점으로 재구성

#### 페이지 3: AI와 함께 설계하기 — 인터뷰 + 보기 제시
- **언제 쓰나**: 뭘 만들지 명확하지 않을 때, 결정이 안 설 때
- **핵심 패턴**:
  - "내 상황 들어보고 질문 5개 해줘" → 답하면서 생각 정리
  - "옵션 3가지로 보기 제시해줘" → 결정 부담 줄이기

#### 페이지 4: 계획서 받고 → 검토 → 진행하기
- **언제 쓰나**: 큰 작업 시키기 전에, AI가 멋대로 가지 못하게
- **핵심 패턴**: "바로 만들지 말고 계획서 먼저 보여줘"
- **연결**: bkit의 `/pdca plan`이 이걸 자동화한 거라는 점

## 5. 작성 형식 — 실전 자연어 위주

### 사용자 결정사항 (중요)

> "이렇게 묻지 마세요 vs 이렇게" 비교박스 ❌
> 실제로 어떻게 자연어로 물어보는지 그대로 ✅

### 페이지 골격

```
1. 언제 이게 필요한가 (1~2문단)
   - 비유나 일상 상황으로 시작
   - "이런 거 막혀본 적 있죠?" 식 공감 진입

2. 가장 단순한 자연어 질문부터
   - 그냥 평소 말하듯 물어보는 예시
   - "이거 안 돼" 수준도 OK

3. 답이 어색할 때 → 더 정교하게 묻는 법
   - 인터뷰 시키기: "내 상황 듣고 질문 해줘"
   - 메타프롬프팅: "이런 거 잘하려면 어떻게 물어봐야 해?"
   - 정교한 프롬프트 받아서 그걸로 다시 대화

4. 실제 대화 예시 한 시나리오 통째로
   - 시나리오 설정 (예: "라이프 위키에 일기 카테고리 추가하고 싶음")
   - 자연어 질문 → AI 답 (요약) → 이어진 대화 → 결과
   - 대화창 형식 박스로 표현

5. 같이 보면 좋은 메타스킬
   - 다른 메타스킬 페이지 링크
```

### 톤 가이드
- 다정한 친구 말투 (기존 환경 세팅 페이지 톤 유지)
- 코드보다 **자연어 대화 예시**가 핵심
- 정교한 프롬프트는 "AI한테 만들어달라고 시키면 됨" 메시지
