# AI 스터디에서 "도구 사용법" 말고 진짜 가르쳐야 할 것

---

1주차 오프모임에서 수강생분들한테 솔직한 얘기를 들었다.

> "전체적으로 이게 어떻게 돌아가는 건지 잘 모르겠어요."
> "모르는 용어가 나오면 그냥 막혀요."
> "결국 뭘 만들게 되는 건지 아직 잘 모르겠어요."

용어 설명 보강, 툴팁 추가, 설치 페이지 추가… 작업하다가 갑자기 깨달았다.

**내가 가장 중요한 걸 사이트에 안 써놨다.**

---

## 말로만 했던 것들

오프모임 중간중간에 나는 계속 이런 말을 했다.

> "막히면 AI한테 물어보세요."
> "AI를 학습 메이트로 쓰세요."
> "AI한테 인터뷰를 시키세요."

수강생들이 고개를 끄덕였다. 그런데 사이트에는 이게 없었다.

나한테 너무 당연해서 안 써놨다. **그게 실수였다.**

---

## 기초와 기본은 다르다

이 스터디를 설계할 때부터 "도구 스터디가 아니다"라고 생각했다.

근데 막상 사이트는 도구 설치 방법, git 사용법, GitHub 세팅으로 채워져 있었다.

이것들은 **기초**다. 기초는 바뀐다.

이 스터디가 처음 시작했을 때 썼던 에디터가 한 달 만에 UI가 통째로 바뀌었다. 그래서 VS Code로 갈아탔다. 기초는 이렇게 바뀐다.

**기본은 다르다.**

"막혔을 때 AI한테 어떻게 물어보는지" — 어떤 도구를 쓰든 똑같이 통한다. 사라지지 않는다. 모든 것의 토대가 된다.

## 이걸 가르치는 곳이 거의 없다

바이브코딩 결과물, 도구 튜토리얼, 프롬프트 잘 쓰는 법 — 이런 건 많다.

"AI랑 어떻게 일하는 습관을 만드는지"를 체계적으로 가르치는 곳은 드물다.

다들 너무 당연해서 그럴 것이다. 나도 그랬다.

근데 수강생 입장에서는 당연하지 않다.

**너무 당연한 거지만 — 이걸 알아야지 뭔가 제대로 해나갈 수 있으니까.**
