# /analyze — 제출물 AI 분석 (강사용)

## 역할

`00_missions/`에 들어온 수강생 사례글을 읽고 강사에게 분석 리포트를 제공합니다.
갤러리 등록 여부 판단, 피드백 초안 작성에 활용합니다.

## 실행 순서

### 1. 분석할 파일 선택

`00_missions/`에서 제출 파일 목록 표시 (`template/` 제외):

```
분석 가능한 제출물:
1. mission-01-홍길동.md
2. mission-01-kimtest.md
```

AskUserQuestion: "어떤 제출물을 분석할까요?"

### 2. 파일 읽기 + 분석

선택한 파일을 읽고 다음 항목을 분석합니다:

**완성도 체크**
- 배포 URL 있는지
- 한 줄 요약이 구체적인지
- 진행 과정이 단계별로 서술됐는지
- 막힌 부분 + 해결 방법이 있는지 (없으면 따라하는 사람에게 가장 아쉬운 부분)
- 인사이트가 개인 경험에서 우러나온 건지 vs 추상적 감상인지

**갤러리 등록 추천 여부**
- 추천 / 보완 후 추천 / 비추천 + 이유

**수강생에게 줄 피드백 초안**
- 잘된 점 1~2가지
- 보완하면 더 좋을 점 1가지 (있을 때만)
- 따뜻하고 짧게 (강사가 직접 보낼 수 있는 톤으로)

### 3. 리포트 출력

```
## 분석 결과: [파일명]

**완성도**: 상 / 중 / 하
**갤러리 등록**: 추천 / 보완 후 추천 / 비추천

### 잘된 점
- ...

### 보완 포인트
- ... (없으면 생략)

### 피드백 초안 (수강생에게 보낼 메시지)
---
[수강생 GitHub 아이디]님, 사례글 잘 읽었습니다.
...
---
```

### 4. 다음 액션 안내

- 갤러리 등록 → `/gallery` 실행
- 피드백 전달 → 위 초안 복사해서 PR 댓글로 사용
