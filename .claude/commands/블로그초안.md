argument-hint: "[소재 키워드 또는 설명 — 생략하면 최근 소재 후보 목록 보여줌]"
---

$ARGUMENTS 값을 확인하라.

**$ARGUMENTS가 비어있으면:**
`journal/20_일간_*.md` 최근 7개 파일에서 `## 소재 후보` 섹션을 모두 읽고
번호 목록으로 보여줘라. 사용자가 어떤 소재를 발전시킬지 고르게 해라.

**$ARGUMENTS가 있으면 또는 사용자가 소재를 골랐으면:**
해당 소재와 관련된 일간 기록 맥락을 참고해서 블로그 초안을 작성하라.

## 블로그 글 작성 규칙

### 프론트매터 (필수)
```yaml
---
title: 글 제목
date: YYYY-MM-DD
projects: [자립]
themes: [키워드1, 키워드2]
status: 초안
description: 한 줄 설명
---
```
status는 반드시 `초안`으로 시작. 사용자가 직접 `공개`로 바꿔야 발행됨.

### 글 작성 규칙
- **첫 줄은 장면으로 시작.** 추상 명제 금지.
- 결론 먼저 내지 말 것. 독자가 같이 발견하게.
- 1000자 내외.
- 단답형 짧은 문장만 연달아 쓰지 마라. 쉼표로 속도 늦추기.
- "결국", "결과적으로", "즉", "다시 말해서" 같은 정리 접속사 자제.
- "이게 진짜 깨달음이었다" 같은 강조 글당 1회.
- 일상 말투 적극: "꼼지락꼼지락", "어찌어찌", "웃긴 게 뭐냐면"
- 클로징은 추상 깨달음 말고 짝맞춤 두세 줄 또는 일상 장면으로 닫기.

### 파일명 규칙
`src/content/blog/[소재-키워드-영문-또는-한글].md`
한글 파일명도 괜찮음.

초안 저장 후 사용자에게 저장 경로 알려줘라.
공개하려면 `status: 초안` → `status: 공개`로 바꾸고 `/publish` 하면 된다고 안내해라.
