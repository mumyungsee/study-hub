> **선택 1~2시간 코스 — 마지막**

## 이번 장에서 할 것

1. 수정한 파일을 GitHub에 Push합니다.
2. Vercel이 자동으로 재빌드하는 것을 확인합니다.
3. 내 사이트에서 반영된 것을 확인합니다.

---

## 1단계. Push하기

채팅창에 입력합니다.

```txt
/project:publish
```

또는 직접 이렇게 요청합니다.

```txt
지금까지 변경된 파일을 확인하고,
문제가 없으면 적절한 커밋 메시지로 커밋한 다음 GitHub에 push해줘.
```

Claude가 변경된 파일을 확인하고 커밋 메시지를 작성한 뒤 Push합니다.

<details>
<summary>Push 오류가 났다면</summary>

```txt
GitHub Push 중 오류가 났어. 오류 메시지 확인하고 해결해줘.
```

</details>

---

## 2단계. Vercel 재빌드 확인하기

Push가 끝나면 Vercel이 자동으로 사이트를 다시 만들기 시작합니다.

브라우저에서 Vercel 대시보드에 들어갑니다.

```txt
https://vercel.com/dashboard
```

내 프로젝트를 클릭하면 빌드 진행 상황이 표시됩니다. `Building` → `Ready`로 바뀌면 완료입니다.

1~2분 후 내 사이트 URL로 들어가서 반영된 것을 확인합니다.

---

## 3단계. 글을 내리고 싶다면

올린 글이 마음에 들지 않으면 `공개`를 `초안`으로 되돌리고 다시 Push합니다.

```txt
[파일명] 글의 status를 다시 초안으로 바꿔줘.
바꾼 다음 publish도 해줘.
```

---

## 튜토리얼을 마치며

여기까지 완료했다면 라이프 위키의 기본 흐름이 갖춰진 것입니다.

```txt
레포 사용 → 바꾸고 싶은 것 발견
  ↓
/plan으로 기획
  ↓
Claude와 수정
  ↓
/project:publish로 배포
```

이 흐름을 계속 반복하면서 내 상황에 맞게 조금씩 바꿔나가면 됩니다.

<details>
<summary>다음에 해볼 수 있는 것들</summary>

**슬래시 명령어 추가하기**

레포에 없는 새 명령어가 필요하다면 Claude에게 이렇게 말합니다.

```txt
.claude/commands/ 폴더에 [명령어 이름].md 파일 만들어줘.
이 명령어는 [어떤 역할]을 하게 해줘.
```

**도메인 연결**

Vercel 대시보드에서 내 도메인을 연결할 수 있습니다. 무료 플랜에서도 가능합니다.

**저장소 이름 바꾸기**

`aaa-starter-kit` 대신 다른 이름으로 바꾸고 싶다면 이렇게 요청합니다.

```txt
내 GitHub 저장소 이름을 life-wiki로 바꾸고,
로컬 설정도 맞게 업데이트해줘.
```

</details>
