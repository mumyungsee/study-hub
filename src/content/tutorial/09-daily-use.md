> **선택 2시간 코스**

## 이번 장에서 할 것

1. 매일 쓰는 흐름을 익힙니다.
2. 글을 비공개로 되돌리는 법을 압니다.

---

## 매일 쓰는 흐름

```txt
/project:daily   → 오늘 회고
/project:blog    → 블로그 초안 (소재가 있을 때만)
/project:publish → GitHub Push → 사이트 자동 업데이트
```

세 줄이 기본 흐름입니다.

---

## 1단계. 오늘 회고 — `/project:daily`

```txt
/project:daily
```

Claude가 대화를 이끌어줍니다. 오늘 있었던 일을 자연스럽게 풀어놓으면 됩니다.

대화가 끝나면 `journal/YYYY-MM-DD.md` 형태로 파일이 저장됩니다.

---

## 2단계. 블로그 초안 — `/project:blog`

회고 안의 소재 후보에서 블로그 글로 발전시키고 싶은 게 있을 때 실행합니다.

```txt
/project:blog
```

초안이 만들어지면 `src/content/blog/` 폴더에 저장됩니다. 기본 status는 `초안`입니다.

공개하고 싶으면 status를 바꿉니다.

```txt
방금 만든 블로그 글 status를 공개로 바꿔줘.
```

---

## 3단계. GitHub에 올리기 — `/project:publish`

```txt
/project:publish
```

Claude가 변경된 파일을 확인하고 커밋 메시지를 작성한 뒤 Push합니다.

Push가 끝나면 Vercel이 자동으로 사이트를 다시 빌드합니다. 1~2분 후 내 사이트에서 반영된 것을 확인할 수 있습니다.

---

## 글을 비공개로 되돌리는 법

올린 글이 마음에 들지 않으면 `공개`를 `초안`으로 되돌리고 다시 Push합니다.

```txt
[파일명] 글의 status를 다시 초안으로 바꿔줘.
바꾼 다음 publish도 해줘.
```

---

<details>
<summary>주간 회고, 월간 회고도 만들고 싶다면</summary>

**주간 회고:**

```txt
/project:weekly
```

최근 7일치 일간 기록을 읽고 주간 회고를 생성합니다. `journal/` 폴더에 저장됩니다.

**월간 회고:**

```txt
/project:monthly
```

이번 달 주간 회고들을 읽고 월간 회고를 생성합니다.

</details>

---

## 이번 장에서 확인한 것

1. 매일 흐름은 `daily → blog → publish` 세 단계입니다.
2. `status: 초안`으로 되돌리고 Push하면 사이트에서 글이 내려갑니다.

---

## 튜토리얼을 마치며

여기까지 완료했다면 라이프 위키의 기본 흐름이 갖춰진 것입니다.

```txt
매일 회고 → 소재 발견 → 블로그 초안 → 공개 → 사이트 업데이트
```

이 흐름을 계속 반복하면서 내 상황에 맞게 조금씩 바꿔나가면 됩니다.

<details>
<summary>다음에 해볼 수 있는 것들</summary>

**도메인 연결**
Vercel 대시보드에서 내 도메인을 연결할 수 있습니다. 무료 플랜에서도 가능합니다.

**저장소 이름 바꾸기**
`aaa-starter-kit` 대신 `life-wiki` 같은 이름으로 바꾸고 싶다면 이렇게 요청합니다.

```txt
내 GitHub 저장소 이름을 life-wiki로 바꾸고,
로컬 설정도 맞게 업데이트해줘.
```

**2-repo 분리**
회고(비공개)와 사이트(공개)를 아예 별개 저장소로 분리하는 구조도 있습니다. 보안을 더 높이고 싶을 때 고려합니다.

</details>
