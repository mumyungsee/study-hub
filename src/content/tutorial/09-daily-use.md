---

## 1단계. `/pdca analyze` 스킬 실행하기

채팅창에 입력합니다.

```txt
/pdca analyze
```

bkit이 Plan 문서와 현재 구현 상태를 비교해서 보고합니다.

```txt
✓ 홈 제목 수정 — 완료
✓ 블로그 카드 스타일 변경 — 완료
✗ 프로필 사진 추가 — 미완료
```

완료된 것과 아직 안 된 것이 나뉘어 표시됩니다.

<details>
<summary>Check 단계에서 스킬이 하는 일</summary>

`/pdca analyze`는 두 가지를 비교합니다.

1. **Plan 문서**: 7장에서 만든 계획 — 무엇을 바꾸기로 했는지
2. **현재 파일 상태**: 실제로 어떻게 바뀌어 있는지

이 비교를 통해 "계획한 것 중 뭐가 됐고 뭐가 안 됐는지"를 정리해줍니다.

`/plan` → `/pdca analyze`로 이어지는 흐름이 bkit의 핵심입니다. 계획을 세우고, 만들고, 비교해서 확인하는 것입니다.

</details>

---

## 2단계. 로컬 미리보기로 눈으로 확인하기

숫자로 확인했으면 실제로 어떻게 보이는지도 봅니다.

```txt
로컬 미리보기 실행해줘.
```

브라우저에서 `http://localhost:4321`로 접속합니다.

- 바꾸고 싶었던 부분이 실제로 바뀌어 있는지 확인합니다.
- 의도하지 않게 망가진 부분이 없는지 확인합니다.

확인이 끝나면 종료합니다.

```txt
로컬 서버 종료해줘.
```

---

## 3단계. 미완료 항목 처리하기

`/pdca analyze` 결과에서 미완료 항목이 있으면 두 가지 중 하나를 선택합니다.

**지금 바로 수정:** 8장으로 돌아가서 추가 수정 후 커밋합니다.

**다음으로 미루기:** 오늘은 여기까지로 정하고 다음 Act 단계에서 기록합니다.

```txt
오늘 못 한 부분은 뭐고, 다음에 하려면 어떻게 하면 될지 정리해줘.
```

---

## 이번 장에서 확인한 것

1. `/pdca analyze` 스킬로 계획 대비 구현 상태를 확인했습니다.
2. 로컬 미리보기로 실제 결과를 눈으로 확인했습니다.
3. 미완료 항목은 지금 처리하거나 다음으로 미룹니다.
