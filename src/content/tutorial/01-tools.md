이 튜토리얼에서는 도구를 다섯 가지 사용합니다.

1. Antigravity
2. Claude Code
3. bkit
4. GitHub
5. Vercel

처음 보면 이름이 많아서 복잡해 보일 수 있습니다. 하지만 역할을 나눠서 보면 단순합니다.

Antigravity와 Claude Code는 내 컴퓨터에서 실제 작업을 도와주는 도구입니다. bkit은 작업을 계획하고 점검하는 흐름을 잡아줍니다. GitHub는 오픈소스 레포를 가져오고 내 작업을 저장하는 곳이고, Vercel은 그 파일을 웹사이트로 배포해줍니다.

## 전체 그림

먼저 큰 흐름을 봅니다.

```txt
GitHub에서 오픈소스 레포 가져오기
  ↓
Antigravity 안에서 Claude Code에게 수정 요청
  ↓
bkit으로 계획과 점검
  ↓
내 GitHub 저장소에 저장
  ↓
Vercel이 웹사이트로 배포
```

이 튜토리얼에서는 `selfishclub/aaa-starter-kit` 레포를 가져와 내 라이프 위키에 맞게 바꾸는 방식으로 진행합니다.

처음부터 빈 폴더에서 모든 것을 만들지 않습니다. 이미 공개된 좋은 구조를 가져와 실행해보고, 내 이름과 내 콘텐츠에 맞게 수정하면서 배웁니다.

## Antigravity

Antigravity는 AI가 내 컴퓨터 안에서 직접 작업할 수 있게 해주는 환경입니다.

일반적인 챗봇은 답변만 해줍니다. 예를 들어 "블로그 글 파일을 만들어줘"라고 말하면 글 내용을 채팅창에 써줄 수는 있지만, 내 폴더에 직접 파일을 만들지는 못합니다.

Antigravity에서는 AI가 실제 프로젝트 폴더를 보고, 파일을 읽고, 새 파일을 만들고, 터미널 작업까지 도와줄 수 있습니다.

이 튜토리얼에서는 Antigravity를 이렇게 사용합니다.

1. 작업 폴더 열기
2. 프로젝트 파일 구조 확인하기
3. AI에게 파일 생성과 수정 요청하기
4. 빌드와 배포 명령 실행 요청하기

초보자는 이렇게 기억하면 됩니다.

> Antigravity는 AI와 함께 내 컴퓨터의 프로젝트를 만지는 작업 공간입니다.

## Claude Code

Claude Code는 Antigravity 안에서 실제로 파일 작업을 도와주는 AI 어시스턴트입니다.

Claude Code에게는 자연어로 요청할 수 있습니다.

예를 들면 이렇게 말합니다.

```txt
이 프로젝트 구조를 확인하고, 초보자가 이해할 수 있게 폴더별 역할을 설명해줘.
```

또는 이렇게 요청할 수 있습니다.

```txt
튜토리얼 페이지를 추가하고, 빌드가 통과하는지 확인한 다음 GitHub에 push해줘.
```

중요한 점은 명령어를 정확히 외워서 치는 것보다 원하는 결과를 구체적으로 말하는 것입니다.

좋은 요청은 보통 세 가지를 포함합니다.

1. 무엇을 하고 싶은지
2. 어떤 파일이나 폴더와 관련이 있는지
3. 끝나고 무엇을 확인해야 하는지

예를 들어 이런 요청이 좋습니다.

```txt
src/content/tutorial 폴더에 1장 도구 소개 문서를 만들어줘.
초보자가 바로 이해할 수 있게 각 도구의 역할과 사용 순간을 설명해줘.
작성 후 npm.cmd run build로 깨지는 곳이 없는지도 확인해줘.
```

초보자는 이렇게 기억하면 됩니다.

> Claude Code는 내가 말로 설명한 작업을 실제 파일과 명령으로 옮겨주는 동료입니다.

## bkit

bkit은 계획, 실행, 점검 흐름을 잡는 데 사용하는 도구입니다.

AI와 작업하다 보면 바로 만들기 시작하고 싶어집니다. 하지만 처음부터 막 만들면 나중에 "지금 뭘 하려고 했지?" 하고 길을 잃기 쉽습니다.

bkit은 이런 상황에서 작업을 단계로 나누는 데 도움을 줍니다.

이 튜토리얼에서는 bkit을 다음 흐름으로 사용합니다.

1. Plan: 무엇을 만들지 계획합니다.
2. Do: 계획한 대로 만듭니다.
3. Check: 제대로 됐는지 확인합니다.
4. Act: 다음에 고칠 점을 반영합니다.

이걸 PDCA라고 부릅니다. 이름을 외울 필요는 없습니다. 그냥 "계획하고, 만들고, 확인하고, 다음에 반영한다"는 흐름만 기억하면 됩니다.

초보자는 이렇게 기억하면 됩니다.

> bkit은 AI와 작업할 때 길을 잃지 않게 해주는 작업 계획표입니다.

## GitHub

GitHub는 코드를 저장하는 곳이면서, 다른 사람이 공개한 오픈소스 프로젝트를 가져와 배울 수 있는 곳입니다.

오픈소스는 누구나 볼 수 있게 공개된 코드와 프로젝트를 말합니다. 어떤 사람은 자기만 쓰려고 만든 도구를 공개하기도 하고, 어떤 팀은 다른 사람이 가져가서 고치고 배울 수 있도록 프로젝트 전체를 열어두기도 합니다.

오픈소스가 좋은 이유는 세 가지입니다.

1. 처음부터 다 만들지 않아도 됩니다.
2. 이미 작동하는 구조를 보면서 배울 수 있습니다.
3. 내 상황에 맞게 고치면서 실력이 늡니다.

이번 튜토리얼에서는 GitHub에 공개된 `selfishclub/aaa-starter-kit` 레포를 가져와 사용합니다.

직접 명령어로 가져오면 이런 형태입니다.

```bash
gh repo clone selfishclub/aaa-starter-kit
```

하지만 이 튜토리얼의 기본 흐름은 명령어를 외워서 직접 치는 방식이 아닙니다. Antigravity에서 Claude Code에게 이렇게 요청해도 됩니다.

```txt
GitHub의 selfishclub/aaa-starter-kit 레포를 이 작업 폴더로 가져와줘.
가져온 뒤 폴더 구조를 확인하고, 라이프 위키로 바꾸려면 어떤 파일을 수정해야 하는지 설명해줘.
```

이 튜토리얼에서 GitHub는 세 가지 역할을 합니다.

1. 오픈소스 레포를 가져오는 출발점입니다.
2. 내 라이프 위키 프로젝트를 저장하고 백업합니다.
3. Vercel이 사이트를 배포할 수 있게 연결 지점이 됩니다.

GitHub에서 자주 나오는 말도 미리 봐둡니다.

1. repository: 프로젝트 저장소
2. commit: 작업 저장 단위
3. push: 내 컴퓨터의 커밋을 GitHub에 올리는 것
4. fork: 다른 사람의 저장소를 내 계정으로 복사하는 것
5. clone: GitHub 저장소를 내 컴퓨터로 가져오는 것

초보자는 이렇게 기억하면 됩니다.

> GitHub는 오픈소스를 가져오고, 내 작업 기록을 저장하고, 배포 서비스와 연결하는 온라인 저장소입니다.

## Vercel

Vercel은 GitHub에 올라간 코드를 웹사이트로 배포해주는 서비스입니다.

우리가 직접 서버를 빌리거나 복잡한 배포 설정을 하지 않아도 됩니다. GitHub 저장소를 Vercel에 연결해두면, main 브랜치에 push할 때마다 Vercel이 자동으로 사이트를 다시 만들어줍니다.

이 튜토리얼에서는 이런 흐름을 사용합니다.

```txt
글 작성
  ↓
status: 공개
  ↓
git commit
  ↓
git push
  ↓
Vercel 자동 배포
```

초보자는 이렇게 기억하면 됩니다.

> Vercel은 GitHub에 올린 프로젝트를 실제 웹사이트로 보여주는 배포 서비스입니다.

## AI에게 이렇게 요청하면 됩니다

처음에는 어떻게 말해야 할지 막막할 수 있습니다. 그럴 때는 아래 문장을 그대로 써도 됩니다.

```txt
이 프로젝트에서 지금 내가 어느 단계에 있는지 확인해줘.
그리고 다음에 해야 할 일을 초보자도 따라 할 수 있게 3단계로 정리해줘.
```

오픈소스 레포를 가져올 때는 이렇게 말합니다.

```txt
selfishclub/aaa-starter-kit 레포를 가져와서 실행 가능한지 확인해줘.
그다음 내 라이프 위키로 바꾸려면 어떤 부분을 수정해야 하는지 초보자 기준으로 설명해줘.
```

배포할 때는 이렇게 말합니다.

```txt
변경된 파일을 확인하고, 문제가 없으면 적절한 커밋 메시지로 커밋한 뒤 GitHub에 push해줘.
Vercel 자동 배포가 시작된다는 것도 알려줘.
```

## 이번 장에서 확인한 것

이번 장에서는 도구 5가지의 역할을 나눠서 봤습니다.

1. Antigravity는 작업 공간입니다.
2. Claude Code는 파일 작업을 도와주는 동료입니다.
3. bkit은 계획과 점검 흐름을 잡아줍니다.
4. GitHub는 오픈소스를 가져오고 내 작업 기록을 저장합니다.
5. Vercel은 웹사이트를 배포합니다.

도구 이름을 전부 외우려고 하지 않아도 됩니다. 지금은 "어느 도구가 어느 순간에 필요한지"만 잡으면 충분합니다.
