이 페이지는 튜토리얼을 따라가다 막혔을 때 펴보는 페이지입니다. 처음부터 다 읽을 필요는 없고, 막힌 부분만 찾아보면 됩니다.

## 화면에서 막힐 때 찾을 메뉴

사이트 UI는 시간이 지나면 바뀔 수 있습니다. 버튼 위치가 달라져도 보통 아래 단어를 찾으면 됩니다.

### GitHub에서 찾을 단어

1. `Sign up`
2. `Sign in`
3. `Settings`
4. `Emails`
5. `Repositories`
6. `New`

### Vercel에서 찾을 단어

1. `Dashboard`
2. `Add New...`
3. `Project`
4. `Import Git Repository`
5. `Import`
6. `Deploy`

### Antigravity에서 찾을 단어

1. `Open Folder`
2. `Open Project`
3. `File`
4. `Terminal`
5. `Chat`

정확히 같은 위치에 없더라도 이 단어들을 기준으로 찾으면 대부분 진행할 수 있습니다.

## 자주 막히는 지점

처음 준비할 때 자주 막히는 부분이 있습니다.

### 이메일 인증을 안 한 경우

GitHub 계정을 만들었지만 이메일 인증을 하지 않으면 저장소 만들기 같은 기본 작업에서 막힐 수 있습니다.

이때는 `프로필 사진 → Settings → Emails`로 들어가 인증 메일을 다시 보냅니다.

### Vercel에서 GitHub 저장소가 안 보이는 경우

Vercel에서 GitHub 저장소가 보이지 않는다면 Vercel이 GitHub 저장소에 접근할 권한을 아직 받지 못한 것일 수 있습니다.

이 경우 Vercel의 `Add New... → Project` 화면에서 GitHub 연결을 다시 확인합니다.

개인 GitHub 계정의 저장소를 Vercel에 연결하려면 보통 그 저장소의 소유자 권한이 필요합니다.

### 명령어가 인식되지 않는 경우

`git`, `gh`, `node`, `npm` 같은 명령어가 인식되지 않으면 두 가지 가능성이 있습니다.

1. 설치가 안 되어 있다 — 해당 도구를 다시 설치합니다.
2. 설치는 됐는데 터미널이 그걸 아직 모른다 — Antigravity를 재시작하거나 새 터미널을 열고 다시 확인합니다.

설치 자체가 헷갈리면 Claude Code에게 이렇게 요청합니다.

```txt
지금 [도구 이름] 명령어가 인식되지 않아.
설치되어 있는지 확인하고, 설치가 안 되어 있으면 내 운영체제에 맞는 방법으로 설치까지 도와줘.
```

### Node.js 알림이 떠서 헷갈리는 경우

Antigravity는 VS Code 기반이라, 작업하다가 "Node.js가 필요합니다" 같은 알림이 뜨는 경우가 있습니다.

알림이 뜨면 그 안내를 그대로 따라가도 됩니다. 보통 공식 다운로드 페이지로 이동시켜주거나, 설치 안내를 줍니다.

알림을 놓쳤거나 안 뜬다면 [2장. 시작 전 준비](/tutorial/02-setup/)의 Node.js 설치 단계로 돌아가 진행하세요.
