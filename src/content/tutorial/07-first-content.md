> **선택 2시간 코스**

## 이번 장에서 할 것

1. `/project:daily`로 오늘 회고를 씁니다.
2. 소재 후보에서 블로그 초안을 만듭니다.
3. 초안을 공개로 바꾸고 로컬에서 확인합니다.

---

## 1단계. 오늘 회고 작성하기

```txt
/project:daily
```

Claude가 오늘 하루를 돌아보는 대화를 시작합니다. 자연스럽게 오늘 있었던 일을 이야기합니다.

대화가 끝나면 `journal/` 폴더에 파일이 생깁니다.

```txt
journal/2026-05-05.md
```

왼쪽 파일 탐색기에서 파일이 생긴 것을 확인합니다.

<details>
<summary>명령어가 목록에 없다면</summary>

```txt
.claude/commands/ 폴더에 어떤 파일이 있는지 확인해줘.
daily 명령어 파일이 있는지도 알려줘.
```

파일이 있는데 목록에 안 보이면 Antigravity를 재시작합니다.

</details>

---

## 2단계. 블로그 초안 만들기

회고 파일 안에 `## 소재 후보` 섹션이 있습니다. 여기서 마음에 드는 소재를 골라 초안을 요청합니다.

```txt
/project:blog
```

또는 직접 이렇게 요청합니다.

```txt
오늘 회고의 소재 후보 중 [소재 이름]을 블로그 글로 써줘.
```

초안이 생기면 `src/content/blog/` 폴더에 파일이 저장됩니다.

---

## 3단계. 공개 설정하기

블로그 글 파일 맨 위를 보면 이렇게 되어 있습니다.

```txt
---
status: 초안
---
```

사이트에 올리려면 `공개`로 바꿉니다.

```txt
방금 만든 블로그 글 status를 초안에서 공개로 바꿔줘.
```

---

## 4단계. 로컬 미리보기로 확인하기

```txt
로컬 미리보기를 실행해줘.
브라우저에서 어떻게 확인하면 되는지도 알려줘.
```

Claude가 서버를 실행합니다. 브라우저에서 `http://localhost:4321`로 접속합니다.

블로그 목록에 방금 쓴 글이 올라와 있는지 확인합니다.

확인이 끝나면 서버를 종료합니다.

```txt
로컬 미리보기 서버 종료해줘.
```

<details>
<summary>터미널에서 직접 실행하는 방법</summary>

```bash
npm install    # 처음 한 번만
npm run dev
```

종료는 `Ctrl + C`를 누릅니다.

</details>

<details>
<summary>글이 사이트에 안 보인다면</summary>

`status: 공개`로 바꿨는지 먼저 확인합니다.

```txt
src/content/blog/ 폴더에 있는 파일들의 status 값을 보여줘.
```

</details>

---

## 이번 장에서 확인한 것

1. `/project:daily`로 회고를 쓰면 `journal/`에 저장됩니다.
2. `/project:blog`로 소재에서 블로그 초안을 뽑습니다.
3. `status: 공개`로 바꾼 글만 사이트에 표시됩니다.
4. 로컬 미리보기로 배포 전에 확인할 수 있습니다.
