> **필수 코스**

## 이번 장에서 할 것

`CLAUDE.md`를 열어서 내 이름, 프로젝트, 언어 설정을 바꿉니다.

---

## 1단계. CLAUDE.md 수정하기

채팅창에 입력합니다.

```txt
CLAUDE.md를 내 상황에 맞게 수정해줘.
내 이름은 [내 이름], 이 프로젝트는 내 일상 회고를 기록하고
블로그 글로 발전시키는 개인 라이프 위키야.
한국어로 대화하는 걸 기본으로 해줘.
```

Claude가 수정 내용을 보여주면 확인하고 승인합니다. 원하지 않는 부분이 있으면 이렇게 말합니다.

```txt
[이 부분]을 [이렇게] 바꿔줘.
```

<details>
<summary>Claude에게 잘 전달되는 요청 방식</summary>

잘 되는 요청에는 세 가지가 포함됩니다.

1. **무엇을** 하고 싶은지
2. **어느 파일**과 관련된 건지
3. **다 되면** 무엇을 확인해야 하는지

예시:

```txt
CLAUDE.md에서 사이트 이름 부분을 "내 라이프 위키"로 바꿔줘.
바꾼 다음 파일 전체 내용을 보여줘.
```

막연하게 "뭔가 바꿔줘"보다 구체적일수록 Claude가 잘 도와줍니다.

</details>

---

## 2단계. 반영 확인하기

```txt
지금 이 프로젝트가 어떤 프로젝트인지 나한테 설명해줘.
```

방금 수정한 내용을 바탕으로 대답하면 잘 적용된 것입니다.

<details>
<summary>CLAUDE.md가 뭔지 궁금하다면</summary>

`CLAUDE.md`는 Claude에게 주는 지침서입니다.

Claude Code는 프로젝트 폴더를 열 때마다 이 파일을 먼저 읽습니다. 여기에 적힌 내용이 이후 대화 전반에 영향을 줍니다. 매번 "내 이름은 OO이고, 이 프로젝트는 이런 구조야"라고 설명하지 않아도 Claude가 처음부터 알고 있게 됩니다.

**지금 바꿔야 할 부분**
- 이름
- 프로젝트 설명
- 언어 설정

**나중에 해도 되는 부분**
- 슬래시 명령어 설명
- 세부 규칙이나 제약 사항

</details>

<details>
<summary>CLAUDE.md 파일이 보이지 않는다면</summary>

```txt
프로젝트 폴더에 CLAUDE.md 파일이 있는지 확인해줘.
없으면 기본 내용으로 새로 만들어줘.
```

</details>

---

## 이번 장에서 확인한 것

1. `CLAUDE.md`에 내 이름과 프로젝트 설명을 넣었습니다.
2. Claude가 바뀐 내용을 인식하는 것을 확인했습니다.
