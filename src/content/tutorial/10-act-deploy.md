> **선택 1~2시간 코스 — 마지막**

## 이번 장에서 할 것

1. 이번 사이클에서 배운 것과 다음에 할 것을 한 줄로 정리합니다.
2. GitHub에 Push합니다.
3. Vercel에서 배포된 것을 확인합니다.

---

## 1단계. Act — 다음 사이클을 위한 한 줄 정리

```txt
이번 작업에서 잘 된 것 한 가지, 다음에 이어서 할 것 한 가지를 정리해줘.
CLAUDE.md 에 간단히 메모로 추가해줘.
```

길게 쓸 필요 없습니다. "다음엔 프로필 사진 추가하기" 한 줄이면 충분합니다.

이 메모가 다음 번 `/plan` 실행할 때 출발점이 됩니다.

<details>
<summary>Act 단계가 왜 필요한가요?</summary>

PDCA에서 Act는 "다음 사이클을 더 잘 돌리기 위한 준비"입니다.

오늘 못 한 것을 기록해두지 않으면 다음에 다시 처음부터 생각해야 합니다. 한 줄이라도 남겨두면 다음 사이클이 빠르게 시작됩니다.

```txt
Plan → Do → Check → Act
                      ↓
               다음 Plan의 출발점
```

</details>

---

## 2단계. GitHub에 Push하기

```txt
/project:publish
```

또는 직접 요청합니다.

```txt
지금까지 변경된 파일을 확인하고,
커밋 메시지는 "act: [이번 작업 요약]"으로 커밋한 다음 GitHub에 push해줘.
```

Push가 끝나면 Vercel이 자동으로 사이트를 다시 빌드합니다.

---

## 3단계. 배포 확인하기

브라우저에서 Vercel 대시보드를 확인합니다.

```txt
https://vercel.com/dashboard
```

`Building` → `Ready`로 바뀌면 완료입니다. 1~2분 후 내 사이트 URL로 접속해서 반영된 것을 확인합니다.

---

## PDCA 한 바퀴 완료

```txt
레포 탐색 (6장)
  ↓
Plan: /plan으로 기획 + 커밋 (7장)
  ↓
Do: Claude와 수정 + 커밋 (8장)
  ↓
Check: /check로 검증 + 로컬 미리보기 (9장)
  ↓
Act: 개선사항 정리 + Push + 배포 (10장)
```

이 사이클을 반복할수록 내 라이프 위키가 점점 내 것이 됩니다.

**슬래시 명령어 추가하기**

새 명령어가 필요하다면 Claude에게 이렇게 말합니다.

```txt
.claude/commands/ 폴더에 [명령어 이름].md 파일 만들어줘.
이 명령어는 [어떤 역할]을 하게 해줘.
```

**도메인 연결**

Vercel 대시보드에서 내 도메인을 연결할 수 있습니다. 무료 플랜에서도 가능합니다.

**저장소 이름 바꾸기**

```txt
내 GitHub 저장소 이름을 life-wiki로 바꾸고,
로컬 설정도 맞게 업데이트해줘.
```

</details>
