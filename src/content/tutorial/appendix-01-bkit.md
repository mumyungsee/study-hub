## bkit이란

bkit(Vibecoding Kit)은 Claude Code 위에 올려진 프롬프트 엔지니어링 프레임워크입니다.

Claude Code 자체는 AI가 파일을 읽고 쓰고 실행하는 도구입니다. bkit은 그 위에 개발 방법론(PDCA + 9단계 파이프라인)과 전문화된 스킬들을 얹어서, 규모 있는 프로젝트도 체계적으로 만들 수 있게 해줍니다.

이 튜토리얼에서 쓴 `/plan`, `/check`가 bkit 스킬입니다.

---

## 주요 스킬

스킬은 Claude Code 채팅창에서 `/스킬이름`으로 실행합니다. 실행하면 bkit이 해당 역할로 전환되어 특정 흐름을 따릅니다.

| 스킬 | 역할 |
|------|------|
| `/plan` | 요구사항 인터뷰 → 계획 문서 생성 (`docs/` 폴더에 저장) |
| `/design` | 계획 문서 → 설계 문서 생성 (컴포넌트 구조, API 설계 등) |
| `/check` | 계획 문서 vs 현재 구현 상태 비교 → 갭 보고 |
| `/learn` | 코드베이스 분석 → 구조와 패턴 학습 문서 생성 |

<details>
<summary>스킬이 일반 대화와 다른 점</summary>

일반 대화는 Claude가 맥락 없이 응답합니다. 스킬은 다릅니다.

`/plan`을 실행하면:

```txt
/plan 입력
  ↓
bkit이 기획 모드로 전환
  ↓
요구사항 인터뷰 (무엇을 만들지 단계별로 질문)
  ↓
계획 문서 자동 생성 (docs/01-plan/ 폴더에 저장)
```

스킬은 `.claude/commands/` 폴더의 마크다운 파일로 정의되어 있습니다. 파일을 열어보면 어떤 지침이 Claude에게 주어지는지 볼 수 있습니다.

</details>

## 9단계 개발 파이프라인

더 복잡한 프로젝트를 만들 때 bkit이 따르는 전체 흐름입니다.

```
PM Analysis  →  Plan  →  Design  →  Do  →  Check  →  Act
```

각 단계가 문서를 남깁니다. 문서가 다음 단계의 입력이 됩니다.

| 단계 | 산출물 | 관련 스킬 |
|------|--------|---------|
| PM Analysis | 요구사항 문서, 사용자 페르소나 | `/pm` |
| Plan | 계획 문서 | `/plan` |
| Design | 설계 문서 (DB 스키마, API, 컴포넌트) | `/design` |
| Do | 실제 구현 | (직접 작업) |
| Check | 갭 분석 보고서 | `/check` |
| Act | 개선 기록 + 다음 사이클 준비 | (직접 기록) |

이 튜토리얼은 Plan → Do → Check → Act 네 단계만 다뤘습니다. 전체 파이프라인은 Dynamic 레벨부터 본격적으로 활용됩니다.

---

## 프로젝트 레벨

bkit은 프로젝트 규모에 따라 세 레벨로 구분합니다.

### Starter

- 오픈소스 레포 커스터마이징
- 간단한 정적 사이트 (지금 이 튜토리얼)
- 주로 쓰는 스킬: `/plan`, `/check`
- 파이프라인: Plan → Do → Check → Act

### Dynamic

- API 연동이 있는 서비스
- 사용자 인증 (로그인/회원가입)
- 데이터베이스 연동
- 주로 쓰는 스킬: `/plan`, `/design`, `/check`
- 파이프라인: PM → Plan → Design → Do → Check → Act

### Enterprise

- 마이크로서비스 아키텍처
- CI/CD 파이프라인
- 보안 요구사항 (OWASP, 인증/인가)
- 대규모 팀 협업
- 전체 9단계 파이프라인

## 언제 다음 레벨로 가나요

다음 상황이 되면 Dynamic을 고려합니다.

- "사용자가 로그인해서 자기 데이터만 보여야 한다"
- "외부 API(날씨, 지도, 결제 등)를 연결해야 한다"
- "서버에 데이터를 저장하고 여러 기기에서 접근해야 한다"

지금 만든 라이프 위키 수준에서는 Starter로 충분합니다.
