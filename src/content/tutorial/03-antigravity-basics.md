> **심화 코스 — 첫 번째**

## 이번 장에서 할 것

수정하기 전에 먼저 써봅니다. 레포를 직접 써봐야 "이 부분을 바꾸고 싶다"는 게 생깁니다.

1. 일간 회고를 써봅니다.
2. 소재에서 블로그 초안을 만들어봅니다.
3. 로컬에서 사이트를 직접 확인합니다.

---

## 1단계. 일간 회고 써보기

채팅창에 입력합니다.

```txt
/project:daily
```

Claude가 오늘 하루를 돌아보는 대화를 시작합니다. 자연스럽게 오늘 있었던 일을 이야기합니다.

대화가 끝나면 `journal/` 폴더에 파일이 생깁니다.

```txt
journal/2026-05-05.md
```

왼쪽 파일 탐색기에서 파일이 생긴 것을 확인합니다.

<details>
<summary>슬래시 명령어가 뭔지 궁금하다면</summary>

`/project:daily`는 **슬래시 명령어**입니다. 매번 긴 요청을 입력하지 않아도 되게 자주 쓰는 지침을 짧게 저장해둔 것입니다.

`.claude/commands/` 폴더 안의 `.md` 파일이 자동으로 슬래시 명령어로 등록됩니다. 파일 안에는 "이 명령어를 실행하면 이렇게 해줘"라는 지침이 적혀 있습니다.

이 레포에는 이미 다섯 가지 명령어가 들어 있습니다.

| 명령어 | 역할 |
|--------|------|
| `/project:daily` | 오늘 하루 회고 대화 → `journal/`에 저장 |
| `/project:blog` | 회고 소재에서 블로그 초안 작성 |
| `/project:weekly` | 최근 7일 일간 기록 → 주간 회고 |
| `/project:monthly` | 이번 달 주간 회고 → 월간 회고 |
| `/project:publish` | 변경 파일 커밋 → GitHub Push |

</details>

<details>
<summary>Antigravity 화면이 낯설다면</summary>

Antigravity 화면은 네 영역으로 나뉩니다.

- **파일 탐색기** (왼쪽): 열어놓은 폴더의 파일 목록
- **에디터** (가운데): 파일 내용을 직접 보고 수정하는 공간
- **채팅창** (오른쪽): Claude와 대화하는 곳 (`Ctrl+L`로 열기)
- **터미널** (아래): Claude가 명령어를 실행할 때 결과가 표시되는 곳

일반 챗봇과 다른 점은, Claude가 답변만 하는 게 아니라 실제로 파일을 만들고 수정한다는 것입니다. 채팅창에서 요청하면 파일 탐색기에 결과가 바로 반영됩니다.

</details>

## 2단계. 블로그 초안 만들어보기

회고 파일 안에 `## 소재 후보` 섹션이 있습니다. 마음에 드는 소재를 골라 초안을 요청합니다.

```txt
/project:blog
```

또는 직접 이렇게 요청해도 됩니다.

```txt
오늘 회고의 소재 후보 중 [소재 이름]을 블로그 글로 써줘.
```

초안이 생기면 `src/content/blog/` 폴더에 파일이 저장됩니다. 기본 status는 `초안`입니다.

사이트에 올리려면 공개로 바꿉니다.

```txt
방금 만든 블로그 글 status를 공개로 바꿔줘.
```

---

## 3단계. 로컬 미리보기로 확인하기

배포 전에 내 컴퓨터에서 먼저 확인합니다.

```txt
로컬 미리보기를 실행해줘.
브라우저에서 어떻게 확인하면 되는지도 알려줘.
```

Claude가 서버를 실행합니다. 브라우저에서 `http://localhost:4321`로 접속합니다.

블로그 목록에 방금 쓴 글이 올라와 있는지 확인합니다.

확인이 끝나면 종료합니다.

```txt
로컬 미리보기 서버 종료해줘.
```

<details>
<summary>글이 사이트에 안 보인다면</summary>

`status: 공개`로 바꿨는지 먼저 확인합니다.

```txt
src/content/blog/ 폴더에 있는 파일들의 status 값을 보여줘.
```

</details>

---

## 이번 장에서 확인한 것

1. `/project:daily`로 회고를 쓰면 `journal/`에 저장됩니다.
2. `/project:blog`로 소재에서 블로그 초안을 뽑습니다.
3. `status: 공개`로 바꾼 글만 사이트에 표시됩니다.
4. 로컬 미리보기로 배포 전에 확인할 수 있습니다.
