> **선택 2시간 코스**

## 이번 장에서 할 것

1. Antigravity 화면 세 영역의 역할을 파악합니다.
2. Claude에게 잘 전달되는 요청 방식을 연습합니다.
3. 슬래시(/) 명령어가 무엇인지 확인합니다.
4. Claude가 파일을 직접 만지는 방식을 체험합니다.

---

## 1단계. 화면 구성 확인하기

```txt
지금 Antigravity에서 열린 폴더 이름이 뭐야?
그리고 그 폴더 안에 어떤 파일이 있는지 목록으로 보여줘.
```

Claude가 파일 목록을 보여주면 폴더가 제대로 연결된 것입니다.

<details>
<summary>화면 영역이 헷갈린다면</summary>

Antigravity 화면은 크게 세 영역으로 나뉩니다.

- **파일 탐색기** (왼쪽): 열어놓은 폴더의 파일 목록
- **채팅창** (가운데/오른쪽): Claude와 대화하는 곳
- **터미널** (아래): Claude가 명령어를 실행할 때 결과가 표시되는 곳

창이 보이지 않으면 Claude에게 물어봅니다.

```txt
파일 탐색기가 안 보여. 어떻게 열면 돼?
```

</details>

---

## 2단계. 잘 전달되는 요청 연습하기

아래 두 가지를 순서대로 입력해봅니다.

짧게 물어보는 방식:

```txt
지금 열린 폴더 안에 어떤 파일이 있어?
```

구체적으로 요청하는 방식:

```txt
README.md 파일을 열어서 첫 줄이 뭔지 보여줘.
```

<details>
<summary>더 잘 전달되는 요청을 만들고 싶다면</summary>

잘 되는 요청에는 세 가지가 들어갑니다.

1. 무엇을 원하는지
2. 어느 파일 또는 폴더와 관련된 건지
3. 다 되면 무엇을 확인해야 하는지

예시:

```txt
src/content/blog/ 폴더에 있는 파일 목록을 보여줘.
각 파일의 status 값이 뭔지도 알려줘.
```

막연하게 "뭔가 바꿔줘"라고 하면 Claude가 추가 확인 질문을 합니다.

</details>

---

## 3단계. 슬래시(/) 명령어 확인하기

채팅창에 `/`를 입력해서 목록이 뜨는지 확인합니다.

지금 목록이 비어 있어도 정상입니다. 명령어는 나중에 직접 만드는 과정이 있습니다.

<details>
<summary>슬래시 명령어가 뭔지 궁금하다면</summary>

슬래시 명령어는 자주 쓰는 요청을 짧게 저장해둔 것입니다. 매번 긴 문장을 입력하지 않아도 됩니다.

이 튜토리얼에서는 나중에 이런 명령어를 직접 만듭니다.

```txt
/project:daily   ← 오늘 회고 시작
/project:blog    ← 블로그 초안 작성
/project:publish ← GitHub에 올리기
```

</details>

## 4단계. Claude가 파일을 직접 만지는 것 체험하기

```txt
지금 열린 폴더에 test.md 파일을 만들어서 "이건 테스트 파일입니다."라고 적어줘.
파일 만든 다음 바로 삭제해줘.
```

왼쪽 파일 탐색기에서 파일이 생겼다가 사라지는 것을 확인합니다.

Claude가 한 일이 마음에 들지 않으면 언제든 이렇게 말합니다.

```txt
방금 한 것 다 되돌려줘.
```

<details>
<summary>일반 챗봇과 뭐가 다른가요?</summary>

일반 챗봇은 답변 텍스트만 돌려줍니다. "파일 만들어줘"라고 해도 내용을 화면에 써줄 뿐, 실제 폴더에 저장하지는 않습니다.

Antigravity에서 Claude는 실제로 파일을 만들고, 읽고, 수정하고, 저장합니다. 작업이 즉시 실제 파일에 반영됩니다.

</details>

---

## 이번 장에서 확인한 것

1. 화면은 파일 탐색기, 채팅창, 터미널 세 영역입니다.
2. 구체적일수록 Claude가 잘 도와줍니다.
3. 슬래시(/) 명령어는 자주 쓰는 요청의 단축키입니다.
4. Claude의 작업은 실제 파일에 바로 반영됩니다.
