이번 장에서는 실제 작업을 시작하기 전에 필요한 준비를 합니다.

아직 코드를 고치지 않습니다. 먼저 계정과 작업 공간을 준비합니다. 초보자에게는 이 단계가 조금 번거롭게 느껴질 수 있지만, 한 번만 해두면 이후 작업이 훨씬 편해집니다.

## 이번 장에서 준비할 것

이번 장의 목표는 네 가지입니다.

1. GitHub 계정 준비
2. Vercel 계정 준비
3. Antigravity에서 작업 폴더 열기
4. GitHub CLI와 git 사용 가능 여부 확인

다 끝나면 다음 장부터 오픈소스 레포를 가져와 실제 라이프 위키로 바꾸는 작업을 시작할 수 있습니다.

## GitHub 계정 만들기

GitHub는 오픈소스 레포를 가져오고, 내 프로젝트를 저장할 곳입니다.

계정이 없다면 먼저 GitHub 계정을 만듭니다.

1. 브라우저에서 `https://github.com`에 들어갑니다.
2. `Sign up`을 누릅니다.
3. 이메일, 비밀번호, 사용자 이름을 입력합니다.
4. 이메일 인증을 완료합니다.

GitHub 공식 문서에서도 GitHub를 사용하려면 무료 개인 계정을 만들고 이메일을 인증해야 한다고 안내합니다.

이미 계정이 있다면 새로 만들 필요는 없습니다. 로그인만 되어 있으면 됩니다.

## GitHub 계정에서 기억할 것

초보자는 GitHub에서 세 가지만 기억하면 됩니다.

1. 내 계정 이름
2. 내가 가져올 오픈소스 레포
3. 내가 새로 만들 라이프 위키 레포

이번 튜토리얼에서 가져올 오픈소스 레포는 이것입니다.

```txt
selfishclub/aaa-starter-kit
```

나중에 내 계정에는 이런 식의 새 저장소를 만들게 됩니다.

```txt
life-wiki
```

이름은 꼭 `life-wiki`일 필요는 없습니다. 하지만 처음 따라 할 때는 짧고 알기 쉬운 이름이 좋습니다.

## Vercel 계정 만들기

Vercel은 GitHub에 올린 프로젝트를 웹사이트로 배포해주는 서비스입니다.

계정이 없다면 Vercel 계정을 만듭니다.

1. 브라우저에서 `https://vercel.com`에 들어갑니다.
2. GitHub 계정으로 가입하거나 로그인합니다.
3. Vercel이 GitHub 저장소에 접근할 수 있도록 연결합니다.

Vercel은 GitHub, GitLab, Bitbucket 같은 Git provider를 연결해서 프로젝트를 가져오는 방식을 지원합니다. 이 튜토리얼에서는 GitHub를 연결합니다.

처음에는 무료 플랜으로 충분합니다.

## 작업 폴더 만들기

이제 내 컴퓨터에 작업할 폴더를 만듭니다.

예를 들어 바탕화면이나 문서 폴더에 이런 폴더를 만들 수 있습니다.

```txt
life-wiki
```

폴더 이름에 한글이나 공백이 있어도 되는 경우가 많지만, 초보자에게는 영어 소문자와 하이픈만 쓰는 이름이 편합니다.

추천 이름:

```txt
life-wiki
```

피하면 좋은 이름:

```txt
내 라이프 위키 최종 진짜최종
```

폴더 이름이 길거나 공백이 많으면 나중에 터미널 명령어를 볼 때 헷갈리기 쉽습니다.

## Antigravity에서 폴더 열기

이제 Antigravity에서 작업 폴더를 엽니다.

흐름은 단순합니다.

1. Antigravity를 실행합니다.
2. 작업 폴더 열기 메뉴를 찾습니다.
3. 방금 만든 `life-wiki` 폴더를 선택합니다.
4. AI 채팅창에서 현재 폴더를 인식하는지 확인합니다.

Antigravity의 화면 구성은 버전에 따라 조금 달라질 수 있습니다. 중요한 것은 "내가 만든 작업 폴더를 Antigravity 안에서 열었다"는 점입니다.

폴더를 연 뒤 Claude Code에게 이렇게 말해봅니다.

```txt
지금 열린 작업 폴더가 어디인지 확인해줘.
그리고 이 폴더가 비어 있는지, 어떤 파일이 있는지 알려줘.
```

비어 있는 폴더라면 괜찮습니다. 다음 장에서 오픈소스 레포를 가져올 것입니다.

## git 설치 확인하기

git은 파일 변경 기록을 관리하는 도구입니다.

직접 설치 여부를 확인하기 어렵다면 Claude Code에게 이렇게 요청합니다.

```txt
이 컴퓨터에 git이 설치되어 있는지 확인해줘.
설치되어 있지 않다면 초보자가 따라 할 수 있게 설치 방법을 안내해줘.
```

직접 터미널에서 확인하고 싶다면 아래 명령어를 사용할 수 있습니다.

```bash
git --version
```

버전 번호가 나오면 설치되어 있는 것입니다.

```txt
git version 2.x.x
```

설치되어 있지 않다면 Git 공식 사이트에서 설치하거나, Antigravity 안에서 Claude Code에게 설치 안내를 요청하면 됩니다.

## GitHub CLI 확인하기

GitHub CLI는 터미널에서 GitHub 작업을 할 수 있게 해주는 도구입니다. 명령어 이름은 `gh`입니다.

이번 튜토리얼에서는 `gh repo clone selfishclub/aaa-starter-kit`처럼 GitHub 레포를 가져오는 명령어를 사용할 수 있습니다.

직접 확인하려면 아래 명령어를 씁니다.

```bash
gh --version
```

설치되어 있으면 버전 번호가 나옵니다.

설치되어 있지 않다면 Claude Code에게 이렇게 말합니다.

```txt
GitHub CLI가 설치되어 있는지 확인해줘.
설치되어 있지 않다면 내 운영체제에 맞는 설치 방법을 안내해줘.
```

설치 후에는 GitHub 로그인이 필요할 수 있습니다. 이때는 보통 아래 명령어를 사용합니다.

```bash
gh auth login
```

GitHub CLI 공식 문서에서도 `gh auth login`으로 GitHub 계정 인증을 진행할 수 있다고 안내합니다.

명령어가 어렵다면 직접 치지 않아도 됩니다. Claude Code에게 이렇게 요청해도 됩니다.

```txt
GitHub CLI 로그인이 필요한지 확인하고, 필요하면 gh auth login 과정을 안내해줘.
```

## 준비 상태 확인 요청하기

여기까지 왔다면 준비가 제대로 되었는지 AI에게 확인시킵니다.

아래 문장을 그대로 사용해도 됩니다.

```txt
지금 이 폴더에서 라이프 위키 튜토리얼을 시작할 준비가 되었는지 확인해줘.
확인할 것은 git 설치 여부, GitHub CLI 설치 여부, 현재 작업 폴더 위치야.
문제가 있으면 초보자가 이해할 수 있게 다음 행동을 알려줘.
```

AI가 확인해줄 때는 보통 이런 정보를 봅니다.

1. 현재 폴더 위치
2. 폴더 안의 파일 목록
3. git 설치 여부
4. GitHub CLI 설치 여부
5. GitHub 로그인 필요 여부

## 자주 막히는 지점

처음 준비할 때 자주 막히는 부분이 있습니다.

### 이메일 인증을 안 한 경우

GitHub 계정을 만들었지만 이메일 인증을 하지 않으면 저장소 만들기 같은 기본 작업에서 막힐 수 있습니다.

GitHub에 로그인한 뒤 이메일 인증 안내가 떠 있는지 확인합니다.

### GitHub와 Vercel 연결이 안 된 경우

Vercel에서 GitHub 저장소가 보이지 않는다면 Vercel이 GitHub 저장소에 접근할 권한을 아직 받지 못한 것일 수 있습니다.

이 경우 Vercel의 계정 설정이나 프로젝트 가져오기 화면에서 GitHub 연결을 다시 확인합니다.

### 명령어가 인식되지 않는 경우

`git` 또는 `gh` 명령어가 인식되지 않으면 설치가 안 되었거나, 설치 후 터미널이 새로 열리지 않은 경우가 많습니다.

이때는 Antigravity를 재시작하거나 새 터미널을 열고 다시 확인합니다.

## 이번 장에서 확인한 것

이번 장에서는 작업을 시작하기 전 필요한 준비를 했습니다.

1. GitHub 계정이 필요합니다.
2. Vercel 계정은 GitHub와 연결해둡니다.
3. 작업 폴더는 짧고 단순한 이름으로 만듭니다.
4. Antigravity에서 작업 폴더를 엽니다.
5. git과 GitHub CLI 설치 여부를 확인합니다.
