## 마크다운이 뭔가요

마크다운(Markdown)은 글을 쓰는 방법 중 하나입니다.

일반 텍스트에 간단한 기호를 붙여서 "이건 제목", "이건 굵게", "이건 코드"라고 표시합니다. 특별한 프로그램 없이 메모장에서도 쓸 수 있고, GitHub·Notion·채팅 앱 등 대부분의 도구에서 읽힙니다.

```
# 제목입니다        →  크게 표시
**굵은 글씨**       →  굵게 표시
- 목록 항목         →  점 목록
```

파일 확장자는 `.md`입니다. 이 튜토리얼의 모든 글, `CLAUDE.md`, 스킬 파일이 전부 마크다운으로 작성되어 있습니다.

## 왜 마크다운인가요

### AI 시대에 특히 중요한 이유

Claude를 포함한 AI는 마크다운을 기본 언어로 씁니다. Claude에게 보내는 지침(CLAUDE.md), Claude가 돌려주는 답변, bkit 스킬 파일이 모두 마크다운입니다.

AI와 대화하면서 자연스럽게 마크다운을 쓰게 됩니다. 문법을 외울 필요는 없지만, "이게 마크다운이구나"를 알면 훨씬 편해집니다.

### 다른 형식과 비교

| | 마크다운 | 워드(docx) | HTML |
|---|---|---|---|
| 파일 형태 | 순수 텍스트 | 바이너리 | 텍스트 |
| 열 수 있는 곳 | 어디서나 | Word 필요 | 브라우저 |
| AI가 읽고 쓰기 | 쉬움 | 어려움 | 복잡함 |
| 버전 관리(Git) | 잘 됨 | 잘 안 됨 | 됨 |

Git으로 변경 이력을 추적하려면 텍스트 파일이어야 합니다. 마크다운이 딱 맞습니다.

## 기본 문법

자주 쓰는 것만 알면 됩니다.

### 제목

```markdown
# 가장 큰 제목 (H1)
## 두 번째 제목 (H2)
### 세 번째 제목 (H3)
```

### 강조

```markdown
**굵게**
*기울임*
~~취소선~~
```

### 목록

```markdown
- 항목 하나
- 항목 둘
  - 들여쓰기 하위 항목

1. 순서 있는 목록
2. 두 번째
3. 세 번째
```

### 링크와 이미지

```markdown
[링크 텍스트](https://example.com)
![이미지 설명](이미지파일.png)
```

### 코드

````markdown
인라인 코드: `코드`

코드 블록:
```javascript
console.log("hello");
```
````

### 표

```markdown
| 열1 | 열2 | 열3 |
|-----|-----|-----|
| A   | B   | C   |
| D   | E   | F   |
```

### 인용

```markdown
> 인용문은 이렇게 씁니다.
```

### 구분선

```markdown
---
```

## 실제로 어디서 쓰나요

### CLAUDE.md 작성

4장에서 수정한 `CLAUDE.md`가 마크다운입니다. 제목(`#`), 목록(`-`), 코드 블록(`` ``` ``)을 섞어서 Claude에게 줄 지침을 구조화했습니다.

### 이 튜토리얼 글들

지금 읽고 있는 이 페이지 포함, 모든 튜토리얼 글이 `.md` 파일입니다. Astro가 마크다운을 읽어서 웹 페이지로 변환합니다.

### GitHub에서

GitHub은 저장소의 `README.md`를 자동으로 렌더링합니다. 마크다운으로 쓴 `README.md`가 프로젝트 소개 페이지가 됩니다.

### Claude와 대화할 때

Claude에게 요청할 때 마크다운을 쓰면 구조가 명확해집니다.

```txt
다음 조건으로 함수를 만들어줘:

**입력**: 사용자 이름 (문자열)
**출력**: 인사말 문자열

- 이름이 비어있으면 "안녕하세요"를 반환
- 이름이 있으면 "안녕하세요, [이름]님"을 반환
```

## 마크다운 파일 열어보기

이 프로젝트의 마크다운 파일을 직접 확인해보려면:

```txt
CLAUDE.md 내용 보여줘.
```

```txt
src/content/tutorial/ 폴더 안에 있는 파일 목록 보여줘.
```

Antigravity 왼쪽 파일 탐색기에서 `.md` 파일을 클릭하면 바로 내용을 볼 수 있습니다.
