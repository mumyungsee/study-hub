이 튜토리얼에서는 AI 도구와 오픈소스 레포를 활용해 나만의 라이프 위키를 만듭니다.

여기서 말하는 라이프 위키는 단순한 블로그가 아닙니다. 매일의 회고를 비공개로 쌓고, 그 안에서 공개할 만한 발견을 골라 블로그 글로 다듬고, GitHub에 올리면 웹사이트까지 자동으로 배포되는 개인 작업 공간입니다.

처음부터 모든 코드를 직접 만들지는 않습니다. 이미 공개된 좋은 구조를 가져와서 실행해보고, 내 이름과 내 프로젝트에 맞게 고치고, 조금씩 이해해갑니다.

## 만들고 나면 이런 모습입니다

완성된 라이프 위키에는 크게 세 가지 공간이 있습니다.

1. 홈
2. 글
3. 튜토리얼

홈은 사이트의 입구입니다. 이 사이트가 어떤 곳인지 소개하고, 글과 튜토리얼로 들어갈 수 있는 길을 보여줍니다.

글 페이지에는 회고에서 발전한 공개 글이 모입니다. 매일의 개인적인 원본 회고는 사이트에 올리지 않고, 공개해도 괜찮은 글만 따로 다듬어 올립니다.

튜토리얼 페이지에는 지금 보고 있는 것처럼 따라 하기용 문서가 장별로 쌓입니다. 나중에는 강의 자료, AI 앱, 도구 같은 메뉴도 추가할 수 있습니다.

## 비공개 기록과 공개 글을 분리합니다

이 튜토리얼에서 가장 중요한 원칙은 비공개 원본과 공개 가공본을 분리하는 것입니다.

매일 회고에는 솔직한 감정, 아직 정리되지 않은 생각, 밖에 공개하고 싶지 않은 내용이 들어갈 수 있습니다. 그래서 회고 파일은 `journal/` 폴더에 저장하고, 사이트에는 노출하지 않습니다.

공개 글은 별도로 `src/content/blog/` 폴더에 저장합니다. 이 폴더에 있는 글 중에서도 `status: 공개`로 표시된 글만 웹사이트에 올라갑니다.

구조를 단순하게 보면 이렇습니다.

```txt
journal/
  20_일간_2026-05-05.md        # 비공개 회고 원본

src/content/blog/
  opensource-repo-nae-geot-mandeulgi.md  # 공개 블로그 글
```

이렇게 나누면 마음 편하게 회고를 쓸 수 있습니다. 모든 기록을 바로 공개해야 한다는 부담이 줄어듭니다.

## 전체 작업 흐름

이 튜토리얼에서 만들 흐름은 다음과 같습니다.

1. 회고를 씁니다.
2. 회고에서 글 소재를 고릅니다.
3. 블로그 초안을 만듭니다.
4. 공개할 글은 `status: 공개`로 바꿉니다.
5. GitHub에 push합니다.
6. Vercel이 자동으로 사이트를 다시 배포합니다.

처음에는 이 흐름이 조금 낯설 수 있습니다. 하지만 한 번 연결해두면 매번 같은 순서로 반복하면 됩니다.

중요한 것은 복잡한 개발 지식을 모두 외우는 것이 아닙니다. "내가 어떤 파일을 고치면 사이트의 어느 부분이 바뀌는지"를 조금씩 익히는 것입니다.

## 이 튜토리얼에서 배우는 것

이 튜토리얼을 따라오면 네 가지를 익히게 됩니다.

1. AI에게 내 컴퓨터의 파일 작업을 맡기는 법
2. 오픈소스 레포를 가져와 내 프로젝트로 바꾸는 법
3. Markdown으로 회고와 블로그 글을 관리하는 법
4. GitHub와 Vercel로 웹사이트를 자동 배포하는 법

여기서 핵심은 "처음부터 완벽하게 이해하고 시작하기"가 아닙니다.

먼저 작동하는 구조를 가져옵니다. 실행해봅니다. 내 이름으로 바꿔봅니다. 글 하나를 넣어봅니다. 에러가 나면 그때 하나씩 해결합니다.

그 과정에서 자연스럽게 GitHub, 배포, 폴더 구조, Markdown, AI 도구 사용법을 익히게 됩니다.

## 우리가 사용할 도구

앞으로 사용할 도구는 다섯 가지입니다.

1. Antigravity
2. Claude Code
3. bkit
4. GitHub
5. Vercel

Antigravity는 AI가 내 컴퓨터 안에서 직접 작업할 수 있게 도와주는 환경입니다. Claude Code는 그 안에서 파일을 읽고 만들고 수정하는 AI 어시스턴트입니다.

bkit은 계획, 실행, 검토 흐름을 잡는 데 사용할 도구입니다. GitHub는 코드를 저장하는 곳이고, Vercel은 GitHub에 올라간 코드를 웹사이트로 바꿔주는 서비스입니다.

지금은 이름만 알고 넘어가도 됩니다. 다음 장에서 하나씩 더 쉽게 풀어봅니다.

## 따라 하기 전에 알아둘 것

이 튜토리얼은 비개발자 또는 AI 도구 입문자를 기준으로 씁니다.

그래서 터미널 명령어를 외우는 방식으로 진행하지 않습니다. 기본 흐름은 Antigravity 안에서 AI에게 자연어로 요청하는 방식입니다.

예를 들면 이렇게 말합니다.

```txt
이 폴더에 있는 프로젝트 구조를 확인하고, 어떤 파일이 어떤 역할을 하는지 초보자도 이해할 수 있게 설명해줘.
```

또는 이렇게 말할 수 있습니다.

```txt
지금까지 만든 변경 사항을 확인하고, 문제가 없으면 커밋하고 GitHub에 올려줘.
```

직접 명령어를 알고 싶은 사람을 위해 중간중간 참고 박스도 넣을 예정입니다. 하지만 이 튜토리얼의 기본 목표는 "명령어를 잘 치는 사람"이 되는 것이 아니라, AI와 함께 작업을 끝까지 굴려보는 것입니다.

## 이번 장에서 확인한 것

이번 장에서는 완성될 라이프 위키의 큰 그림을 봤습니다.

1. 라이프 위키는 회고, 블로그, 튜토리얼이 연결된 개인 작업 공간입니다.
2. 비공개 회고와 공개 글은 폴더부터 분리합니다.
3. 공개 글은 `status: 공개`인 것만 사이트에 올라갑니다.
4. GitHub에 push하면 Vercel이 자동 배포합니다.
5. 처음부터 다 만들기보다 공개된 구조를 가져와 내 것으로 바꿉니다.
